## 0) Правило приоритета (норматив)
1) `КАНОН.md` — единственный канон (при конфликте верить ему).

## 1) INDEX как реестр структуры (норматив)
INDEX — единственный реестр структуры проекта: список файлов, назначение, владельцы, точки входа сборки, связи `\input{...}`, статусы.

**Ответственный за INDEX:** A (Аксиоматика).  
B/C изменения структуры (новый файл, переименование, новый `\input`, перенос раздела) делают только через заявку A:
- что изменилось (1–3 строки)
- имя файла(ов)
- где подключено (в каком `.tex`, какой `\input{...}`)
- статус (draft / candidate / canon)

## 6) Соглашения об обозначениях (единый стиль)
- `d_A`, `l`, `⟨l⟩`; СЭ/НЭ/ЭВО/ГП/КП/ВР; `χ ∈ {−1,+1}`.
- КУБК: Φ только как **интенсивность контактов на Σ**.
- Прокси: (K,G,A_ij) только как **измерительные ярлыки** (запрещена причинная лексика).
- Тесты: отличать “V=0” (вектор) и “V_∥=0” (скаляр).
- Лексика (норматив для отчётов/текстов): запрещено писать “фронт ВР”; допускается только “распространяющаяся/непрерывная возмущённость (ВР)”.
- Термин “образ ЭВО”: использовать как объект описания через Σ + прокси/контактную статистику/Δp (без “полевой” причинности).
Если ведётся LaTeX:
- макросы: `\SE`, `\NE`, `\EVO`, `\GP`, `\KP`, `\VR`, `\NY`, `\A`.
